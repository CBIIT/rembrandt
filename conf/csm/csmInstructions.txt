Steps for setting up CSM:

************************************

1. Modify ApplicationSecurityConfig.xml line 21 in "WEB_ROOT/CSM/" to change your JBoss Path
	eg:
	<hibernate-config-file> 
  		X:/MY_JBOSS_DIR/server/default/deploy/rembrandt.war/csm
	</hibernate-config-file>
 
*************************************
2. Go to MY_JBOSS/server/default/deploy/properties-service.xml
	 and add the attribute tag listed below INSIDE the tag that looks like the following:
    <mbean code="org.jboss.varia.property.SystemPropertiesService"...: 
   
    <attribute name="Properties">
   		gov.nih.nci.security.configFile=MY_JBOSS_DIR/server/default/deploy/rembrandt.war/csm/ApplicationSecurityConfig.xml
    </attribute>

   IMPORTANT: Make sure you modify the "MY_JBOSS_DIR" to point to your JBoss installation.

*************************************
    
3. Place oracle-ds.xml in the jboss-4.0.2\server\default\deploy directory

We actually aren't using this right now so don't worry about all the settings missing


*************************************

4. Add the following lines to jboss-4.0.2\server\default\conf\login-config.xml

    <application-policy name="rembrandt">
		<authentication>
			<login-module code="gov.nih.nci.security.authentication.loginmodules.LDAPLoginModule" flag="required">
				<module-option name="ldapHost">ldaps://ncids2b.nci.nih.gov:636</module-option>
				<module-option name="ldapSearchableBase">ou=nci,o=nih</module-option>
				<module-option name="ldapUserIdLabel">cn</module-option>
			</login-module>			
		</authentication>
	</application-policy>
